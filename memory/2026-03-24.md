## Key Concepts Explained

### COLMAP (Structure-from-Motion)
1. **Feature Extraction**: SIFT keypoints detection
2. **Feature Matching**: Match features across image pairs
3. **Sparse Reconstruction**: Estimate camera poses and 3D points
4. **Bundle Adjustment**: Refine camera poses and 3D structure

### Gaussian Splatting
- Represents 3D scenes as collection of 3D Gaussians (ellipsoids)
- Each Gaussian has: position, scale, rotation, color (SH), opacity
- Training via differentiable rendering with photometric loss
- Loss = 0.8 * L1 + 0.2 * SSIM

### Photometric Loss
- Measures difference between rendered and ground truth images
- L1: Pixel-wise absolute difference
- SSIM: Structural similarity (perceptual quality)
- Drives optimization to match input video frames

## Pipeline Flow Summary

```
User Upload → FastAPI → MinIO Storage → Celery Queue → GPU Worker
  ↓
Frame Extraction (FFmpeg) → Frame Intelligence (Blur/Motion)
  ↓
COLMAP (Camera Poses) → MiDaS (Depth Maps)
  ↓
Gaussian Splatting (7000 iterations) → Optimization (Prune/LOD)
  ↓
Tiling (Octree) → MinIO Storage → Streaming API
  ↓
Frontend (Three.js) → Progressive Tile Loading → Custom Shader Rendering
```

## Lessons Learned

1. **Photogrammetry is complex**: COLMAP handles camera pose estimation, but requires good video quality and feature matches
2. **Gaussian Splatting is powerful**: Creates high-quality 3D models from video, but computationally expensive
3. **Progressive loading is essential**: Tiling and LOD enable smooth browser rendering of large scenes
4. **Pipeline is modular**: Each stage can be optimized independently
